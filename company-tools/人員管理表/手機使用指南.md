# 📱 人員排班系統 - 手機使用指南

## 🌐 方法一：部署到網頁伺服器（推薦）

### 使用 GitHub Pages（免費）

1. **上傳到 GitHub**
   - 在 GitHub 建立新的 Repository
   - 將 `index.html`、`style.css`、`script.js` 三個檔案上傳

2. **啟用 GitHub Pages**
   - 進入 Repository 的 Settings
   - 找到 Pages 選項
   - Source 選擇 `main` 分支
   - 儲存後會得到網址，例如：`https://你的帳號.github.io/專案名稱`

3. **分享給手機用戶**
   - 將網址傳給需要使用的人
   - 他們可以在手機瀏覽器中打開
   - 建議加入手機桌面書籤以便快速存取

### 使用 Netlify（免費，更簡單）

1. 前往 [netlify.com](https://www.netlify.com/)
2. 註冊/登入帳號
3. 將整個資料夾拖曳到 Netlify 上傳
4. 自動生成網址，例如：`https://random-name-123.netlify.app`
5. 可以自訂網址名稱

## 📂 方法二：本地網路分享

### 使用 Python 簡易伺服器

1. **在電腦上開啟命令提示字元（cmd）**
   ```bash
   cd "C:\Users\Jayter\給我自己用的\我的專案區\company-tools\人員管理表"
   ```

2. **啟動簡易伺服器**
   ```bash
   python -m http.server 8000
   ```

3. **查詢電腦 IP 位址**
   ```bash
   ipconfig
   ```
   找到「IPv4 位址」，例如：`192.168.1.100`

4. **手機連線**
   - 確保手機與電腦連接同一個 Wi-Fi
   - 在手機瀏覽器輸入：`http://192.168.1.100:8000`
   - 即可開啟排班系統

**注意**：關閉命令提示字元視窗會停止伺服器

## 📲 方法三：直接傳送檔案

1. **打包檔案**
   - 將 `index.html`、`style.css`、`script.js` 三個檔案壓縮成 ZIP

2. **傳送到手機**
   - 透過 Line、Email、雲端硬碟等方式傳送
   - 手機解壓縮後，用瀏覽器打開 `index.html`

**優點**：不需要網路連線即可使用
**缺點**：資料無法同步，每個人有自己的資料

---

## 🎯 建議使用方式

### ✅ 最佳方案：GitHub Pages 或 Netlify
- **優點**：
  - 永久網址，隨時存取
  - 自動 HTTPS 加密
  - 免費且穩定
  - 更新容易（只需上傳新檔案）

### ⚠️ 注意事項
- 此系統使用瀏覽器的 localStorage 儲存資料
- **每個裝置的資料是獨立的**，不會自動同步
- 建議定期使用「匯出資料」功能備份
- 需要同步資料時，使用「匯入資料」功能

## 📱 手機操作說明

### 手機版優化功能

1. **折疊面板**
   - 「時段選擇」預設折疊，點擊標題可展開
   - 「任務池」預設折疊，點擊標題可展開
   - 節省螢幕空間，方便操作

2. **人員卡片**
   - 單欄顯示，卡片更大
   - 方便拖移任務到人員卡片上
   - 字體放大，更容易閱讀

3. **複製排班表**
   - 格式已優化：按人員列出
   - 顯示每個人 0-24 點的任務
   - 移除階級和分機資訊
   - 適合直接貼到 Line、Telegram 群組

### 建議瀏覽器
- iOS：Safari
- Android：Chrome
- 確保瀏覽器版本為最新

## 🔄 資料同步方式

由於目前沒有後端資料庫，建議使用以下方式同步資料：

1. **主管理者**（通常是電腦端）
   - 排好班後，點擊「匯出資料」
   - 下載 JSON 檔案

2. **其他使用者**（手機或其他裝置）
   - 收到 JSON 檔案後，點擊「匯入資料」
   - 選擇檔案即可同步最新資料

3. **排班表分享**
   - 使用「預覽排班表」功能
   - 點擊「複製文字」
   - 直接貼到通訊軟體群組

## ❓ 常見問題

**Q: 為什麼手機看不到任務池？**
A: 手機版任務池預設折疊，點擊「[任務池]」標題即可展開。

**Q: 如何把任務分配給人員？**
A: 展開任務池，長按任務卡片，拖移到人員卡片上即可。

**Q: 資料會遺失嗎？**
A: 資料儲存在瀏覽器中，除非清除瀏覽器資料，否則不會遺失。建議定期匯出備份。

**Q: 可以多人同時編輯嗎？**
A: 目前不支援即時同步，需要透過「匯出/匯入」功能手動同步資料。
