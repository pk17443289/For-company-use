# 🚀 快速開始指南

這份文件將帶您在 **10 分鐘內** 完成 PT 打卡系統的基本設定。

## ⏱️ 第一步：建立 Google Sheets（3 分鐘）

1. **開啟 Google Sheets**
   - 前往 https://sheets.google.com
   - 點選「空白」建立新試算表
   - 將試算表命名為「PT打卡系統」

2. **建立三個工作表**
   - 將「工作表1」重新命名為「員工資料」
   - 點選左下角「+」新增工作表，命名為「打卡記錄」
   - 再新增一個工作表，命名為「通知設定」

3. **設定「員工資料」工作表**
   ```
   在第一列輸入以下標題：
   A1: 員工編號
   B1: 姓名
   C1: 部門
   D1: 電話
   E1: 建立時間
   ```

4. **設定「打卡記錄」工作表**
   ```
   在第一列輸入以下標題：
   A1: 日期
   B1: 時間
   C1: 員工編號
   D1: 員工姓名
   E1: 部門
   F1: 類型
   G1: 時間戳記
   ```

5. **設定「通知設定」工作表**
   ```
   在第一列輸入以下標題：
   A1: 設定項目
   B1: 設定值
   ```

6. **設定共用權限**
   - 點選右上角「共用」
   - 將「一般存取權」改為「知道連結的任何人」
   - 權限設為「編輯者」
   - 點選「完成」

7. **複製試算表 ID**
   - 從瀏覽器網址列複製試算表 ID
   - 格式：`https://docs.google.com/spreadsheets/d/{這裡就是ID}/edit`

✅ **完成！Google Sheets 已準備就緒**

---

## 🔑 第二步：取得 API 金鑰（4 分鐘）

1. **前往 Google Cloud Console**
   - 開啟 https://console.cloud.google.com/

2. **建立新專案**
   - 點選頂部的專案選擇器
   - 點選「新增專案」
   - 專案名稱：PT打卡系統
   - 點選「建立」

3. **啟用 Google Sheets API**
   - 在搜尋欄輸入「Google Sheets API」
   - 點選搜尋結果中的「Google Sheets API」
   - 點選「啟用」按鈕

4. **建立 API 金鑰**
   - 點選左側「憑證」
   - 點選「建立憑證」→「API 金鑰」
   - 複製產生的 API 金鑰
   - 點選「關閉」

5. **（建議）限制 API 金鑰**
   - 點選剛建立的 API 金鑰
   - API 限制：選擇「限制金鑰」
   - 勾選「Google Sheets API」
   - 點選「儲存」

✅ **完成！API 金鑰已取得**

---

## ⚙️ 第三步：設定系統（2 分鐘）

1. **複製設定檔範本**
   - 在專案資料夾中找到 `config.example.js`
   - 複製並重新命名為 `config.js`

2. **編輯 config.js**
   - 用文字編輯器開啟 `config.js`
   - 將 `YOUR_API_KEY_HERE` 替換為您的 API 金鑰
   - 將 `YOUR_SPREADSHEET_ID_HERE` 替換為您的試算表 ID

   ```javascript
   const CONFIG = {
       API_KEY: '貼上您的API金鑰',
       SPREADSHEET_ID: '貼上您的試算表ID',
       // 其他設定保持預設即可
   };
   ```

3. **儲存檔案**

✅ **完成！系統設定已完成**

---

## 🌐 第四步：開啟系統（1 分鐘）

1. **開啟 index.html**
   - 用瀏覽器開啟 `index.html` 檔案
   - 推薦使用 Chrome 或 Edge 瀏覽器

2. **測試系統**
   - 如果看到「員工管理」頁面，表示成功！
   - 如果顯示錯誤，請檢查 config.js 的設定

✅ **完成！系統已可使用**

---

## 🎯 第五步：新增第一位員工

1. **填寫員工資料**
   ```
   姓名：測試員工
   員工編號：E001
   部門：測試部門
   電話：0912-345-678
   ```

2. **點選「新增員工並生成 QR Code」**
   - 系統會顯示該員工的 QR Code
   - 您可以下載或列印 QR Code

3. **測試打卡功能**
   - 開啟 `attendance.html`
   - 點選「開始掃描」
   - 掃描剛才產生的 QR Code
   - 點選「上班打卡」

4. **檢視打卡記錄**
   - 開啟 `admin.html`
   - 您應該可以看到剛才的打卡記錄

✅ **恭喜！您已成功完成所有設定**

## 📱 使用流程

### 管理者日常操作

1. **新增員工**（index.html）
   - 填寫員工資料
   - 生成並提供 QR Code 給員工

2. **查看記錄**（admin.html）
   - 查看即時統計
   - 篩選查詢打卡記錄
   - 匯出報表

### 員工日常操作

1. **打卡**（attendance.html）
   - 掃描個人 QR Code 或手動輸入員工編號
   - 點選「上班打卡」或「下班打卡」
   - 確認打卡成功

## 💡 小技巧

   - Chrome：右鍵點選分頁 → 釘選分頁

## 🆘 遇到問題？

### 常見錯誤

**「無法載入員工資料」**
- 檢查 API_KEY 是否正確
- 檢查 SPREADSHEET_ID 是否正確
- 確認試算表權限已開啟
- 確認工作表名稱正確（包含大小寫）

**「打卡後沒有記錄」**
- 確認試算表權限設為「編輯者」
- 檢查 Google Sheets API 是否已啟用
- 檢查瀏覽器的開發者工具（F12）Console 是否有錯誤

**「QR Code 掃描器無法開啟」**
- 確認瀏覽器已允許相機權限
- 確認網站使用 HTTPS（或在本地端測試）
- 嘗試使用手動輸入功能

### 需要更多協助？

- 📖 詳細說明：閱讀 `README.md`
- 📊 試算表設定：閱讀 `SHEETS_SETUP.md`
- 🔧 檢查設定：確認 `config.js` 所有值是否正確
